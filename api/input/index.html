input

Lightweight wrapper over Roblox's InputAction / InputContext / InputBinding instances. Lets you declare input contexts as nested tables of actions, register them once, then enable or disable whole contexts on demand.

The model is: bindings (key combinations) live on actions (named, typed inputs like "jump" or "move"), which live on contexts (collections of actions enabled together, e.g. "gameplay" vs "menu"). A context is enabled when its enable count is positive; input.enable / input.disable increment and decrement that count.

See: alicesaidhi/383b04a42cad53acb27e017f50fd8086.

action​

</>
input.action(
v: Action--

An action descriptor with a type discriminator.

) → ActionTypes

Reconciles an action descriptor into an InputAction instance and returns it augmented with the ActionFunctions API (is_pressed, is_released, is_changed, get_state, plus pressed, released, and state_changed signals).

The return type is narrowed by the type field on the descriptor: "bool" → ActionBool, "1d" → Action1D, etc.

binding​

</>
input.binding(v: Binding) → Binding

Identity helper that returns its argument typed as a [Binding]. Useful for declaring bindings inline with type inference.

clear_state​

</>
input.clear_state() → ()

Clears the per-frame transient state (is_pressed, is_released, is_changed flags) for every action. Call this once per frame before polling so each press/release/state-change is observed exactly once.

capture_input​

This is a yielding function. When called, it will pause the Lua thread that called the function until a result is ready to be returned, without interrupting other scripts. Yields
</>
input.capture_input() → {
bind: Enum.KeyCode,
modifier1: Enum.KeyCode,
modifier2: Enum.KeyCode
}

Yields until the user produces input, then returns a table containing the primary KeyCode along with up to two modifier keys held during the press. Mouse buttons and touch are translated to their KeyCode equivalents.
