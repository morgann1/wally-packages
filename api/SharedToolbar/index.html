SharedToolbar

Lets multiple Studio plugins cooperatively own a single PluginToolbar. Peers discover each other through a shared BindableEvent medium parented under StudioService, elect a leader (lowest sessionId wins), and the leader builds the toolbar with every peer's buttons mirrored onto it. Clicks, active state, and enabled state are forwarded to the originating peer over the same medium.

Why: installing several plugins that each call plugin:CreateToolbar with the same name produces duplicate toolbars in Studio. Sharing the toolbar via this module collapses them into one.

RegisterOptions​

</>
type RegisterOptions = {
plugin: Plugin,
mediumName: string,
toolbarTitle: string,
peerId: string,
protocolVersion: number?,
buttons: {ButtonSpec}
}

Options passed to SharedToolbar.register.

  • plugin — your plugin instance.
  • mediumName — a name shared by every cooperating plugin; the BindableEvent parented under StudioService uses this as its name.
  • toolbarTitle — the title to advertise. The peer with the lowest peerId wins, so authoritative naming is stable across reshuffles.
  • peerId — a stable identifier for this plugin (e.g. its name).
  • protocolVersion — optional override for the wire protocol version.
  • buttons — list of buttons to contribute.

register​

</>
SharedToolbar.register(options: RegisterOptions) → SharedToolbarHandle

Joins (or creates) a shared toolbar identified by options.mediumName and contributes the buttons in options.buttons to it. Returns a SharedToolbarHandle for managing this peer's button state.

Cleans up automatically when options.plugin unloads.
