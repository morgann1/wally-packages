OwnedToolbar

Wraps plugin:CreateToolbar with a registry that lets you create or reuse buttons by id and keeps each button's icon in sync with the current Studio theme. Plain values in, no Fusion state objects.

Pair with [useToolbarButton] from React if you want declarative button lifecycle inside a component tree.

Creates a new plugin toolbar and returns an [OwnedToolbarApi] for registering buttons. Pass the plugin instance and the dev-env name used by IsLocalDev (a "Dev" marker button is added when matching).

</>
OwnedToolbar.useToolbarButton(
toolbar: OwnedToolbarApi,
props: ToolbarButtonProps
) → PluginToolbarButton?

React hook that ensures toolbar has a button matching props for the lifetime of the calling component. Reuses an existing button by id, applies active / clickableWhenViewportHidden updates, and wires props.onClick through the button's Click signal.

Returns the live PluginToolbarButton instance, or nil until the first effect commit.
