WideLogger

A logger built around "wide events" (https://loggingsucks.com/): one structured event per operation, enriched with context as the operation runs, emitted exactly once when the span finishes.

Instead of scattering print calls across a code path, you open a span at the start of an operation, attach high-cardinality context fields (user ids, request ids, business state) along the way, and let the logger emit a single readable record at the end. 

local Log = require(path.to.WideLogger)

Log.withSpan("checkout.process", { request_id = id }, function(span)
    span:set("user_id", user.Id)
    span:set("cart_total_cents", cart.totalCents)
    -- ... do work; if this errors the span auto-fails ...
    span:set("payment_provider", "stripe")
end)

Output goes to print (ok) or warn (failed) and is formatted for Studio readability: 

[checkout.process] OK 142.30ms
  cart_total_cents = 4200
  payment_provider = "stripe"
  request_id = "abc"
  user_id = 5

WideLogger.withSpan() ŌåÆ┬Ā()

Open a span, run fn(span), and finish the span automatically. If fn errors, the span is marked failed and the error is rethrown after the event is emitted. Returns whatever fn returned on success.

WideLogger:fail() ŌåÆ┬Ā()

Mark the span as failed and attach the given error. The span will be emitted via warn when finished. Returns the span for chaining.
